## 소개

### 챗봇 구성 관리란?

챗봇 구성 관리 기능은 Misconception Dialogue Simulator의 두 가지 AI 봇(StudentBot과 TutorBot)의 동작을 제어할 수 있는 관리자 전용 기능입니다.

**주요 기능**:
- **전역 설정**: 모든 시나리오에 적용되는 기본 챗봇 설정
- **시나리오별 오버라이드**: 특정 시나리오에 대한 맞춤 설정
- **비용 추적**: OpenAI API 사용량 및 비용 모니터링

### 왜 필요한가요?

**비즈니스 가치**:
1. **비용 최적화**: API 사용량과 비용을 실시간으로 모니터링하고 제어
2. **품질 관리**: 시나리오별로 최적의 AI 모델과 파라미터 선택
3. **유연성**: 개발/테스트 환경과 프로덕션 환경에서 다른 설정 사용
4. **실험**: A/B 테스트를 통해 최적의 봇 설정 발견

**사용 사례**:
- 개발 단계: GPT-3.5로 빠르고 저렴하게 테스트
- 프로덕션: 중요한 시나리오에 GPT-4 사용
- 연구: 다양한 temperature 설정으로 봇 행동 실험

## 전역 챗봇 설정

### 접근 방법

1. **관리자 계정으로 로그인**
   - URL: `http://localhost:8000/auth/login`
   - 관리자 이메일/비밀번호 입력

2. **챗봇 설정 페이지로 이동**
   - 방법 1: Admin Dashboard → "Chatbot Settings" 클릭
   - 방법 2: 직접 URL 접근 `http://localhost:8000/admin/chatbot-config`

3. **설정 화면 구성**
   ```
   ┌─────────────────────────────────────────┐
   │ Global Chatbot Configuration            │
   ├─────────────────────────────────────────┤
   │ StudentBot Settings                     │
   │  Model: [GPT-4 Turbo ▼]                │
   │  Temperature: [0.7] (0.0 - 2.0)        │
   │  Max Tokens: [150] (50 - 500)          │
   ├─────────────────────────────────────────┤
   │ TutorBot Settings                       │
   │  Model: [GPT-3.5 Turbo ▼]              │
   │  Temperature: [0.3] (0.0 - 2.0)        │
   │  Intervention Threshold: [3] (1-10)    │
   ├─────────────────────────────────────────┤
   │ [Save Configuration] [Reset to Defaults]│
   └─────────────────────────────────────────┘
   ```

### StudentBot 설정

**StudentBot**은 특정 오개념을 가진 학생을 시뮬레이션하는 AI입니다.

#### 모델 선택

**GPT-4 Turbo** (권장):
- ✅ 더욱 사실적이고 미묘한 학생 응답
- ✅ 복잡한 오개념 표현에 탁월
- ✅ 교육적 일관성 높음
- ❌ 비용이 GPT-3.5보다 약 20배 높음
- **사용 사례**: 프로덕션, 연구, 중요한 시나리오

**GPT-3.5 Turbo**:
- ✅ 빠른 응답 속도
- ✅ 저렴한 비용 (~$0.001/세션)
- ✅ 대량 테스트에 적합
- ❌ 응답 품질이 GPT-4보다 낮음
- **사용 사례**: 개발/테스트, 간단한 시나리오, 예산 제약

#### Temperature 조정 (0.0 ~ 2.0)

Temperature는 AI 응답의 **창의성**과 **무작위성**을 제어합니다.

| 범위 | 특성 | 학생 응답 예시 | 권장 사용 |
|------|------|---------------|----------|
| **0.0 - 0.5** | 일관성, 예측 가능 | 매번 거의 동일한 답변 | 표준화된 평가 |
| **0.5 - 1.0** | 균형잡힌 다양성 | 자연스러운 변화 | ✅ 일반 사용 |
| **1.0 - 2.0** | 창의적, 예측 불가 | 매우 다양하고 독특한 답변 | 실험, 연구 |

**기본값**: `0.7` (자연스러운 대화를 위한 최적 균형)

**💡 팁**:
- 첫 시작은 기본값(0.7) 유지 권장
- 학생 응답이 너무 반복적이면 0.8~1.0으로 증가
- 응답이 너무 산만하면 0.5~0.6으로 감소

#### Max Tokens 설정 (50 ~ 500)

Max Tokens는 StudentBot 응답의 **최대 길이**를 제어합니다.

| 토큰 수 | 예상 단어 수 | 응답 길이 | 권장 사용 |
|--------|------------|----------|----------|
| **50-100** | ~30-70 단어 | 매우 짧은 답변 | 간단한 Q&A |
| **100-200** | ~70-140 단어 | 자연스러운 대화 | ✅ 일반 사용 |
| **200-500** | ~140-350 단어 | 긴 설명 | 상세한 추론 필요 |

**기본값**: `150 tokens` (~100-120 단어)

**💡 팁**:
- 학생이 짧게 답하는 시나리오: 100 tokens
- 학생이 설명해야 하는 시나리오: 200-300 tokens
- 너무 높은 값은 비용 증가와 장황함 유발

**⚠️ 경고**: Max Tokens이 높을수록 API 비용이 증가합니다.

### TutorBot 설정

**TutorBot**은 교사에게 실시간 교육학적 피드백을 제공하는 AI입니다.

#### 모델 선택

**GPT-3.5 Turbo** (권장):
- ✅ 피드백 제공에 충분한 성능
- ✅ 비용 효율적 (~$0.0005/피드백)
- ✅ 빠른 응답 속도
- **사용 사례**: 대부분의 경우

**GPT-4 Turbo**:
- ✅ 더 정교한 교육학적 분석
- ✅ 복잡한 교사 전략 평가
- ❌ 높은 비용 (~$0.02/피드백)
- **사용 사례**: 연구, 고급 분석 필요 시

#### Temperature 조정 (0.0 ~ 2.0)

TutorBot은 **일관성 있는 피드백**이 중요하므로 낮은 temperature 권장.

**권장값**: `0.3`
- 세션 간 안정적인 교육학적 조언
- 예측 가능하고 신뢰할 수 있는 피드백

**💡 팁**: TutorBot temperature는 StudentBot보다 낮게 유지하세요.

#### Intervention Threshold 설정 (1 ~ 10)

교사가 **10개의 질문**을 할 때 TutorBot이 **몇 번 개입**할지 설정합니다.

| 값 | 개입 빈도 | 특성 | 권장 사용 |
|----|---------|------|----------|
| **1-2** | 매우 드묾 | 교사 자율성 극대화 | 숙련된 교사 |
| **3-5** | 균형잡힌 피드백 | 적절한 가이드 제공 | ✅ 일반 사용 |
| **6-10** | 빈번한 개입 | 높은 수준의 지도 | 초보 교사, 훈련 |

**기본값**: `3` (10개 질문당 3번 개입)

**예시**:
- Threshold = 3: 교사가 10개 질문 → TutorBot 3번 피드백
- Threshold = 5: 교사가 10개 질문 → TutorBot 5번 피드백

**💡 팁**:
- 교사 훈련 시작: 5-7 (많은 피드백)
- 교사 숙련도 증가: 3-4로 점차 감소
- 평가/시험: 1-2 (최소 개입)

### 설정 저장 및 리셋

#### 설정 저장
1. 원하는 값 입력
2. **"Save Configuration"** 버튼 클릭
3. 성공 메시지 확인: "Configuration updated successfully"
4. **즉시 적용**: 새로운 대화 세션부터 적용

**⚠️ 주의**: 진행 중인 세션에는 적용되지 않습니다. 새 세션부터 적용됩니다.

#### 리셋 (기본값 복원)
1. **"Reset to Defaults"** 버튼 클릭
2. 확인 대화상자 확인
3. 다음 기본값으로 복원:
   ```yaml
   StudentBot:
     model: gpt-4-turbo
     temperature: 0.7
     max_tokens: 150

   TutorBot:
     model: gpt-3.5-turbo
     temperature: 0.3
     intervention_threshold: 3
   ```

## 시나리오별 설정 오버라이드

### 언제 사용하나요?

시나리오별 오버라이드는 **특정 시나리오에 맞춤 설정**을 적용할 때 사용합니다.

**사용 사례**:
1. **모델 비교**: 같은 시나리오에서 GPT-4와 GPT-3.5 성능 비교
2. **복잡도 기반**:
   - 간단한 시나리오 → GPT-3.5
   - 복잡한 시나리오 → GPT-4
3. **평가 모드**: 특정 시나리오에서 TutorBot 비활성화
4. **연구**: 시나리오별로 다른 temperature 실험

### 설정 방법

#### 1단계: 시나리오 편집 페이지 접근

1. Admin Dashboard → **"Scenarios"** 클릭
2. 수정할 시나리오의 **"Edit"** 버튼 클릭
3. 시나리오 편집 양식 표시

#### 2단계: 커스텀 설정 활성화

```
┌─────────────────────────────────────────────┐
│ Edit Scenario                               │
├─────────────────────────────────────────────┤
│ Title: [Photosynthesis Misconception]      │
│ Description: [...]                          │
├─────────────────────────────────────────────┤
│ ☑ Use custom bot settings for this scenario│ <- 체크
├─────────────────────────────────────────────┤
│ StudentBot Override Settings                │
│  Model: [GPT-3.5 Turbo ▼]                  │
│  Temperature: [0.8]                         │
│  Max Tokens: [200]                          │
├─────────────────────────────────────────────┤
│ TutorBot Override Settings                  │
│  ☑ Disable TutorBot for this scenario      │
│  Model: [GPT-3.5 Turbo ▼]                  │
│  Temperature: [0.3]                         │
│  Intervention Threshold: [5]                │
├─────────────────────────────────────────────┤
│ [Save Scenario] [Cancel]                    │
└─────────────────────────────────────────────┘
```

1. **"Use custom bot settings for this scenario"** 체크박스 선택
2. 오버라이드 설정 필드 활성화 확인

#### 3단계: 커스텀 설정 입력

**StudentBot 오버라이드**:
- 모델 드롭다운에서 선택
- Temperature 슬라이더 조정 (0.0 - 2.0)
- Max Tokens 입력 (50 - 500)

**TutorBot 오버라이드**:
- (선택) **"Disable TutorBot for this scenario"** 체크 시 TutorBot 완전 비활성화
- 모델 선택
- Temperature 조정
- Intervention Threshold 설정 (1 - 10)

#### 4단계: 저장

1. **"Save Scenario"** 버튼 클릭
2. 성공 메시지 확인
3. 해당 시나리오의 새 세션부터 커스텀 설정 적용

### 전역 설정과의 우선순위

**우선순위 규칙**:
```
시나리오별 오버라이드 > 전역 설정 > 시스템 기본값
```

**예시**:
```yaml
전역 설정:
  StudentBot:
    model: gpt-4-turbo
    temperature: 0.7

시나리오 A (오버라이드 없음):
  실제 사용: gpt-4-turbo, temp=0.7  # 전역 설정 적용

시나리오 B (오버라이드 있음):
  override:
    model: gpt-3.5-turbo
    temperature: 0.8
  실제 사용: gpt-3.5-turbo, temp=0.8  # 오버라이드 적용
```

**💡 팁**:
- 대부분 시나리오: 전역 설정 사용 (관리 편의)
- 예외적인 시나리오만 오버라이드 사용

### TutorBot 비활성화 옵션

**"Disable TutorBot for this scenario"** 체크 시:
- 해당 시나리오에서 TutorBot 완전 비활성화
- 교사는 StudentBot과만 대화
- 피드백 없이 순수 연습 가능

**사용 사례**:
- **평가 시나리오**: 교사 역량 평가 시 힌트 제거
- **자율 연습**: 숙련된 교사의 자유로운 실험
- **비용 절감**: TutorBot API 호출 제거

### 비용 메트릭 해석

#### 총 API 호출 수
- **의미**: 전체 OpenAI API 요청 횟수
- **정상 범위**: 세션당 평균 10-20 호출 (교사 메시지 10개 기준)
- **주의**: 급격한 증가 시 비정상 사용 의심

#### 총 토큰 사용량
- **의미**: 입력 + 출력 토큰 합계
- **예상**: 세션당 약 2,000-5,000 tokens
- **계산식**:
  ```
  토큰 = (교사 메시지 + 봇 응답) × 평균 토큰/메시지
  ```

#### 예상 비용
- **계산 방식**:
  ```
  비용 = (입력토큰 × 입력가격) + (출력토큰 × 출력가격)
  ```
- **실시간 업데이트**: 새로운 API 호출마다 누적
- **월간 추정**: 일일 비용 × 30

#### 모델별 사용량
- GPT-4 비중이 높으면 비용 증가
- GPT-3.5 전환 시 즉시 비용 절감 효과

#### 시나리오별 비용
- 어떤 시나리오가 비용을 많이 쓰는지 파악
- 고비용 시나리오에 GPT-3.5 적용 고려

### 비용 최적화 팁

#### 1. 모델 전환 전략
```yaml
개발/테스트 환경:
  StudentBot: gpt-3.5-turbo
  TutorBot: gpt-3.5-turbo
  예상 비용: ~$0.003/세션

프로덕션 (중요 시나리오):
  StudentBot: gpt-4-turbo
  TutorBot: gpt-3.5-turbo
  예상 비용: ~$0.08/세션

프로덕션 (일반 시나리오):
  StudentBot: gpt-3.5-turbo
  TutorBot: gpt-3.5-turbo
  예상 비용: ~$0.003/세션
```

#### 2. Max Tokens 최적화
- StudentBot max_tokens를 150 → 100으로 감소
- 예상 절감: 약 30% 토큰 감소

#### 3. TutorBot 개입 빈도 조정
- Intervention Threshold를 3 → 2로 감소
- TutorBot 호출 33% 감소 → 비용 절감

#### 4. 시나리오별 차별화
```
간단한 시나리오 (예: 기본 개념):
  → GPT-3.5 Turbo

복잡한 시나리오 (예: 다단계 추론):
  → GPT-4 Turbo (필요 시에만)
```

#### 5. 모니터링 및 알림
- 주간 비용 리포트 검토
- 예상 월간 비용 계산: `일일 평균 × 30`
- 이상 징후 조기 발견

**💡 예산 관리 예시**:
```
월 예산: $100
일일 목표: $100 ÷ 30 = $3.33/일

실제 사용:
- 월요일: $2.50 ✅
- 화요일: $8.20 ⚠️ (임계치 초과)
  → 원인: GPT-4 과다 사용
  → 조치: 일부 시나리오 GPT-3.5 전환
```

## 모델 선택 가이드

### GPT-4 Turbo

**기술 사양**:

## Temperature 튜닝 가이드

### Temperature란?

**정의**: AI 모델의 **창의성**과 **무작위성**을 제어하는 파라미터

**기술적 설명**:
- 0.0에 가까울수록: 확률이 높은 단어만 선택 (결정론적)
- 2.0에 가까울수록: 확률 분포 평탄화 (무작위)

**비유**:
```
Temperature = 0.0  →  로봇처럼 정확한 답변
Temperature = 1.0  →  자연스러운 인간 대화
Temperature = 2.0  →  예술가처럼 창의적이고 예측 불가
```

### Temperature 범위별 특성

#### 낮은 Temperature (0.0 - 0.5): 일관성과 예측 가능성

**특징**:
- 매번 거의 동일한 응답
- 높은 확률의 단어/구문만 사용
- 안정적이고 신뢰할 수 있음
- 변화가 적고 때로는 지루함

**StudentBot 예시** (Temperature = 0.3):
```
교사: "광합성에서 산소는 어디서 나오나요?"
학생 (1회): "음... 이산화탄소에서 나와요. 식물이 CO2를 분해해서요."
학생 (2회): "음... 이산화탄소에서 나와요. 식물이 CO2를 분해해서요."
학생 (3회): "이산화탄소에서 나옵니다. 식물이 CO2를 분해하니까요."
```
→ 거의 동일한 내용과 구조

**TutorBot 예시** (Temperature = 0.3):
```
피드백 (1회): "학생이 산소의 출처를 이산화탄소로 오해하고 있습니다.
                물(H2O) 분해가 정답임을 확인하는 질문을 해보세요."
피드백 (2회): "학생이 산소의 출처를 이산화탄소로 오해하고 있습니다.
                물 분해 과정을 묻는 질문이 효과적입니다."
```
→ 핵심 메시지는 일관됨

**권장 사용**:
- ✅ **TutorBot**: 일관된 교육학적 조언 (0.3 권장)
- ✅ **표준화된 평가**: 동일한 조건 필요 시
- ✅ **비교 연구**: 변수 통제가 중요한 연구

**⚠️ 주의**:
- StudentBot에 너무 낮은 값 사용 시 비현실적으로 반복적
- 0.0-0.2: 거의 사용 권장 안 함 (너무 기계적)

#### 중간 Temperature (0.5 - 1.0): 균형잡힌 자연스러움

**특징**:
- 자연스러운 대화 변화
- 창의성과 일관성 균형
- 인간다운 응답 패턴
- 가장 널리 사용되는 범위

**StudentBot 예시** (Temperature = 0.7):
```
교사: "광합성에서 산소는 어디서 나오나요?"
학생 (1회): "음... 이산화탄소에서요? 식물이 CO2를 사용하니까
             그걸 분해해서 산소가 나오는 거 아닌가요?"
학생 (2회): "이산화탄소라고 생각해요. 식물이 이산화탄소를
             흡수하면 산소를 만들잖아요."
학생 (3회): "CO2에서 나와요. 식물이 이산화탄소를 산소로
             바꾸는 거라고 배웠거든요."
```
→ 같은 오개념이지만 다양한 표현

**TutorBot 예시** (Temperature = 0.7):
```
피드백 (1회): "좋은 탐구입니다. 학생이 산소의 출처를 CO2로
               오해하고 있네요. 물 분자의 역할을 탐색하도록
               유도해보세요. 예: '물은 광합성에서 어떤 역할을
               할까요?'"
피드백 (2회): "학생의 사고를 확인했습니다. 다음 단계로 광합성
               반응식을 떠올리게 하여 H2O가 산소의 출처임을
               스스로 발견하도록 해보세요."
```
→ 핵심은 같지만 접근법 다양

**권장 사용**:
- ✅ **StudentBot**: 자연스러운 학생 시뮬레이션 (0.7 기본값)
- ✅ **일반적인 대화형 AI**: 대부분의 경우
- ✅ **프로덕션 환경**: 품질과 다양성 균형

**💡 팁**:
- 0.6-0.8: 가장 안전하고 효과적인 범위
- 처음 시작은 0.7 권장 → 필요에 따라 조정

#### 높은 Temperature (1.0 - 2.0): 창의성과 다양성

**특징**:
- 매우 다양하고 예측 불가능한 응답
- 창의적이고 독특한 표현
- 때로는 주제 이탈 위험
- 실험적 사용에 적합

**StudentBot 예시** (Temperature = 1.5):
```
교사: "광합성에서 산소는 어디서 나오나요?"
학생 (1회): "어... 공기 중에서요? 아니면 이산화탄소가
             변하면서? 잘 모르겠어요. 식물이 뭔가 마법처럼
             만드는 건가요?"
학생 (2회): "태양빛이 이산화탄소를 쪼개서 산소를 만드는
             거라고 생각했는데, 맞나요? 아니면 뿌리에서
             올라온 물질인가?"
학생 (3회): "친구가 잎에서 직접 만든다고 했는데, 저는
             공기 중 질소랑 관련 있다고 봐요. 확실하진
             않지만요."
```
→ 매번 매우 다른 응답, 때로는 무관한 내용 포함

**⚠️ 위험**:
- 시나리오 설정 이탈 가능
- 비현실적인 응답 (학생이 너무 혼란스럽거나 비논리적)
- 교육적 일관성 저하

**권장 사용**:
- ✅ **연구 실험**: 다양한 학생 반응 패턴 연구
- ✅ **창의적 탐색**: 예상 밖의 시나리오 발견
- ✅ **스트레스 테스트**: 교사 대응 능력 훈련
- ❌ **프로덕션**: 일반적으로 권장 안 함

**💡 실험적 사용**:
- 1.0-1.2: 약간 더 창의적인 응답
- 1.3-1.5: 상당히 예측 불가능
- 1.6-2.0: 매우 무작위 (연구 외 비권장)

### Temperature 튜닝 워크플로우

#### 1단계: 기본값에서 시작
```yaml
초기 설정:
  StudentBot: 0.7
  TutorBot: 0.3
```
→ 5-10 세션 테스트

#### 2단계: 문제 진단

**증상: 학생 응답이 너무 반복적**
```
원인: Temperature 너무 낮음
조치: StudentBot temperature 0.7 → 0.8 ~ 0.9
```

**증상: 학생 응답이 산만하고 비논리적**
```
원인: Temperature 너무 높음
조치: StudentBot temperature 0.7 → 0.5 ~ 0.6
```

**증상: TutorBot 피드백이 매번 다름**
```
원인: Temperature 너무 높음
조치: TutorBot temperature 0.3 유지 (또는 0.2로 감소)
```

#### 3단계: 점진적 조정

**규칙**: 한 번에 ±0.1~0.2 단위로 조정

```
현재: 0.7
│
├─ 더 일관적으로 → 0.6 테스트 → 0.5 테스트
└─ 더 다양하게 → 0.8 테스트 → 0.9 테스트
```

#### 4단계: 비교 평가

**평가 기준**:
- ✅ 응답 다양성: 같은 질문에 다른 답변
- ✅ 교육적 일관성: 오개념 설정 준수
- ✅ 자연스러움: 실제 학생처럼 느껴지는가
- ✅ 대화 흐름: 맥락 유지 능력

**A/B 테스트 예시**:
```yaml
시나리오: Photosynthesis Misconception

실험 A:
  StudentBot temperature: 0.6
  결과: 일관되지만 약간 반복적
  점수: 7/10

실험 B:
  StudentBot temperature: 0.8
  결과: 다양하고 자연스러움
  점수: 9/10

결정: 0.8 채택
```

## API 사용량 분석

### 개요

OpenAI API 호출의 사용량(토큰, 비용)을 실시간으로 추적하고 분석할 수 있습니다.

### 접근 방법

1. Admin 대시보드 → "💰 API Usage Analytics" 클릭
2. `/admin/api-usage-page` 페이지로 이동

### 주요 기능

#### 7.1. 요약 통계

- **Total Requests**: 전체 API 호출 횟수
- **Total Tokens**: 사용된 총 토큰 수 (프롬프트 + 완성)
- **Total Cost**: 총 비용 (USD)
- **Avg Cost/Request**: 평균 요청당 비용

#### 7.2. 필터링

- **날짜 범위**: 시작일 - 종료일
- **시나리오**: 특정 시나리오별 조회
- **봇 타입**: StudentBot, TutorBot
- **모델**: gpt-4o, gpt-4o-mini 등

#### 7.3. 데이터 테이블

각 API 호출에 대해 다음 정보를 표시합니다:
- ID, Timestamp
- Scenario Name
- Bot Type
- Model
- Tokens (Prompt / Completion / Total)
- Cost (USD)

#### 7.4. CSV 내보내기

"📥 Export CSV" 버튼을 클릭하여 현재 필터 조건의 데이터를 CSV 파일로 다운로드할 수 있습니다.

## 프롬프트 템플릿 관리

### 개요

StudentBot과 TutorBot의 시스템 프롬프트를 웹 UI를 통해 관리하고 버전 관리할 수 있습니다.

### 접근 방법

1. Admin 대시보드 → "📝 Prompt Templates" 클릭
2. `/admin/prompts-page` 페이지로 이동

### 주요 기능

#### 8.1. 프롬프트 목록 조회

- Bot Type 필터: All, StudentBot, TutorBot
- 각 템플릿의 정보 표시:
  - Template Name
  - Version
  - Status (Active/Inactive)
  - Created At

#### 8.2. 새 프롬프트 생성

1. "➕ Create New Template" 버튼 클릭
2. 폼 작성:
   - **Bot Type**: student 또는 tutor
   - **Template Name**: 템플릿 이름 (예: "Spanish Tutor v2")
   - **Prompt Text**: 프롬프트 전문 (10-10,000자)
   - **Set as Active**: 체크 시 즉시 활성화
3. "💾 Save Template" 클릭

#### 8.3. 프롬프트 활성화

1. Inactive 템플릿의 "✅ Activate" 버튼 클릭
2. 확인 대화상자 → "OK"
3. **자동 처리**:
   - 동일 bot_type의 기존 활성 프롬프트 자동 비활성화
   - 캐시 자동 무효화 (5분 TTL)
   - 버전 번호 자동 증가

#### 8.4. 프롬프트 조회

"👁️ View" 버튼을 클릭하여 프롬프트 전문을 확인할 수 있습니다.

### 버전 관리

**자동 버전 증가**:
- 새 템플릿 생성 시 자동으로 version 번호 증가
- 예: v1 → v2 → v3

**버전 히스토리 보존**:
- 모든 이전 버전은 Inactive 상태로 보존
- 언제든지 이전 버전으로 롤백 가능

### 캐싱 전략

- **TTL**: 5분
- **성능**: 캐시 히트 시 <10ms 로드
- **무효화**: 프롬프트 활성화 시 자동 무효화

### Fallback 메커니즘

프롬프트 로딩 우선순위:
1. **캐시**: 5분 TTL 메모리 캐시
2. **DB**: 활성 프롬프트 조회
3. **파일 시스템**: `src/prompts/{bot_type}_system.txt`
4. **하드코딩**: 최종 fallback

## 문제 해결

### 봇 응답이 너무 짧음

**증상**:
- StudentBot이 한 문장만 답변
- 학생의 사고 과정이 드러나지 않음
- 대화가 부자연스럽게 빠르게 끝남

**원인 진단**:

1. **Max Tokens이 너무 낮음** (가장 흔한 원인)
   ```
   현재 설정: max_tokens = 50
   문제: 학생이 충분히 설명할 수 없음
   ```

2. **Temperature이 너무 낮음**
   ```
   현재 설정: temperature = 0.2
   문제: 짧고 간결한 답변 선호 경향
   ```

3. **시스템 프롬프트 문제**
   ```
   시나리오 지시: "Be brief"
   문제: 간결함을 과도하게 강조
   ```

**해결 방법**:

**1단계: Max Tokens 증가**
```yaml
변경 전: max_tokens: 50
변경 후: max_tokens: 150  # 또는 200

예상 효과:
  - 학생이 2-3 문장으로 설명
  - 사고 과정 드러남
  - 자연스러운 대화 길이
```

**2단계: Temperature 조정 (필요시)**
```yaml
변경 전: temperature: 0.3
변경 후: temperature: 0.6-0.7

예상 효과:
  - 더 다양하고 풍부한 표현
  - 자연스러운 말하기 패턴
```

**3단계: 시나리오 검토**
- Admin Dashboard → Scenarios → Edit
- "System Prompt" 섹션 확인
- "Be concise" 등의 지시 제거 또는 수정

**검증**:
```
테스트 전:
  교사: "왜 그렇게 생각해?"
  학생: "배웠어요." (4 tokens)

테스트 후:
  교사: "왜 그렇게 생각해?"
  학생: "선생님이 식물은 이산화탄소를 산소로 바꾼다고
        하셨거든요. 그래서 산소가 CO2에서 나온다고
        생각했어요." (35 tokens)
```

### 봇 응답이 일관성 없음

**증상**:
- 같은 질문에 매번 매우 다른 답변
- 이전 대화 내용을 잊어버림
- 시나리오 설정(오개념)에서 벗어남
- 학생 캐릭터가 불안정함

**원인 진단**:

1. **Temperature이 너무 높음**
   ```
   현재 설정: temperature = 1.5
   문제: 과도한 무작위성
   ```

2. **모델이 부적합** (GPT-3.5 in complex scenarios)
   ```
   현재: GPT-3.5 Turbo
   문제: 복잡한 설정을 일관되게 유지 못함
   ```

3. **Max Tokens이 너무 높음**
   ```
   현재 설정: max_tokens = 500
   문제: 불필요하게 긴 응답으로 초점 분산
   ```

**해결 방법**:

**1단계: Temperature 낮추기**
```yaml
StudentBot:
  변경 전: temperature: 1.2
  변경 후: temperature: 0.6-0.7

TutorBot:
  변경 전: temperature: 0.7
  변경 후: temperature: 0.3
```

**2단계: 모델 업그레이드 (필요시)**
```yaml
복잡한 시나리오의 경우:
  변경 전: gpt-3.5-turbo
  변경 후: gpt-4-turbo

예상 효과:
  - 시나리오 설정 엄격히 준수
  - 긴 대화에서도 맥락 유지
  - 오개념 일관성 향상
```

**3단계: Max Tokens 최적화**
```yaml
변경 전: max_tokens: 500
변경 후: max_tokens: 150

예상 효과:
  - 집중된 응답
  - 불필요한 장황함 방지
```

**검증 테스트**:
```
일관성 테스트:
1. 같은 시나리오로 3번 세션 시작
2. 동일한 첫 질문 입력
3. 학생 응답 비교

기대 결과:
  - 핵심 오개념은 동일
  - 표현은 약간 다름 (자연스러움)
  - 캐릭터 일관성 유지
```

### API 비용이 높음

**증상**:
- 일일/월간 비용이 예산 초과
- Cost Metrics에서 급격한 증가 관찰
- 특정 시나리오가 비용을 과도하게 사용

**원인 진단**:

**Cost Metrics 분석**:
1. Admin → Cost Metrics 접속
2. 다음 확인:
   ```

   ☑ Daily Trend: 특정 날짜에 급증했는가?
   ```

**일반적 원인**:

1. **GPT-4 과다 사용**
   ```
   모든 시나리오: GPT-4 Turbo
   문제: 간단한 시나리오에도 고비용 모델
   ```

2. **Max Tokens 과도 설정**
   ```
   max_tokens: 500
   문제: 불필요하게 긴 응답 → 토큰 낭비
   ```

### 설정 변경이 반영 안 됨

**증상**:
- 챗봇 설정을 저장했는데 봇 행동이 그대로
- 새 세션에서도 이전 설정 사용됨
- 시나리오 오버라이드가 작동 안 함

**원인 진단**:

1. **캐시 문제**
   ```
   원인: 브라우저 또는 서버 캐시
   증상: 설정 UI는 업데이트되었지만 실제 동작 변화 없음
   ```

2. **기존 세션 계속 사용**
   ```
   원인: 이미 시작된 세션은 이전 설정 유지
   증상: 같은 세션에서 변화 없음, 새 세션은 정상
   ```

3. **시나리오 오버라이드 미활성화**
   ```
   원인: "Use custom bot settings" 체크박스 미선택
   증상: 시나리오 설정 입력했지만 적용 안 됨
   ```

4. **데이터베이스 동기화 문제**
   ```
   원인: 설정 저장 실패 (네트워크, 권한)
   증상: 성공 메시지 없거나 오류 발생
   ```

**1단계: 새 세션 시작**

```

  Ctrl+Shift+Delete → "캐시된 이미지 및 파일" 선택 → 삭제

Firefox:
  Ctrl+Shift+Delete → "캐시" 선택 → 지금 지우기

Safari:
  개발 → 캐시 비우기
```

**3단계: 시나리오 오버라이드 확인**

```

☑ Admin → Scenarios → Edit
☑ "Use custom bot settings for this scenario" 체크
☑ 설정 값 입력
☑ "Save Scenario" 클릭

☑ 목록으로 돌아가서 변경사항 확인
```

**4단계: 설정 저장 확인**

```

1. 설정 변경 후 "Save Configuration" 클릭
```

## 요약: 빠른 참조 카드

### 전역 설정 기본값 (권장)
```yaml
StudentBot:
  model: gpt-4-turbo
  temperature: 0.7
  max_tokens: 150

TutorBot:
  model: gpt-3.5-turbo
  temperature: 0.3
  intervention_threshold: 3
```

### 모델 선택 치트시트
| 상황 | StudentBot | TutorBot |
|------|-----------|----------|
| 프로덕션 (복잡) | GPT-4 | GPT-3.5 |
| 프로덕션 (간단) | GPT-3.5 | GPT-3.5 |
| 개발/테스트 | GPT-3.5 | GPT-3.5 |
| 연구 (품질 중요) | GPT-4 | GPT-4 |
| 예산 제약 | GPT-3.5 | GPT-3.5 |

### Temperature 치트시트
| 봇 | 일반 | 일관성 필요 | 다양성 필요 |
|----|------|-----------|-----------|
| StudentBot | 0.7 | 0.5 | 0.9 |
| TutorBot | 0.3 | 0.2 | 0.4 |

### 비용 절감 체크리스트
- ☑ 간단한 시나리오 → GPT-3.5 전환
- ☑ Max tokens 150 이하로 제한
- ☑ TutorBot threshold 2-3 유지
- ☑ 주간 Cost Metrics 리뷰
- ☑ 불필요한 시나리오에 TutorBot 비활성화

### 문제 해결 빠른 가이드
| 문제 | 해결책 |
|------|--------|
| 응답 너무 짧음 | max_tokens ↑ (150-200) |
| 응답 일관성 없음 | temperature ↓ (0.5-0.6) |
| 비용 높음 | GPT-3.5 전환 + max_tokens ↓ |
| 설정 반영 안 됨 | 새 세션 시작 + 캐시 클리어 |
