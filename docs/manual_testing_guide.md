# 수동 테스트 가이드: 메시지 즉시 표시 기능

## 목적
Wave 2에서 구현한 메시지 즉시 표시 기능의 수동 테스트 체크리스트입니다.

## 사전 준비
1. 서버 실행: `uvicorn src.main:app --reload`
2. 브라우저에서 http://localhost:8000 접속
3. 학생 로그인 (student_uid: test_student)

## 기본 기능 테스트

### 1. 서버 실행 확인
- [ ] http://localhost:8000 접속 시 로그인 페이지 표시
- [ ] 콘솔 에러 없음

### 2. 로그인 및 세션 생성
- [ ] 학생 UID와 닉네임으로 로그인 성공
- [ ] 시나리오 선택 페이지로 이동
- [ ] 시나리오 선택 후 대화 인터페이스 표시

### 3. 메시지 전송 후 즉시 표시 확인 (핵심 기능)
- [ ] **메시지 입력 후 전송 버튼 클릭**
- [ ] **학생 메시지가 0.1초 이내에 화면에 표시됨**
- [ ] **챗봇 응답이 1-2초 후에 화면에 표시됨**
- [ ] **각 메시지에 타임스탬프 표시**

### 4. 입력 필드 자동 초기화 확인
- [ ] 메시지 전송 후 입력 필드가 즉시 비워짐
- [ ] 전송 버튼이 즉시 비활성화됨 (disabled)
- [ ] 새로운 메시지 입력 가능 상태로 전환

### 5. 폴링 시스템 작동 확인
- [ ] 메시지 전송 후 2초마다 새 메시지 확인 (개발자 도구 Network 탭 확인)
- [ ] `/sessions/{session_id}/messages/updates?since={message_id}` 엔드포인트 호출 확인
- [ ] 새 메시지가 없으면 빈 응답 반환

### 6. 중복 메시지 표시 없음 확인
- [ ] 동일한 메시지가 두 번 표시되지 않음
- [ ] `data-message-id` 속성으로 메시지 구분 확인 (개발자 도구)
- [ ] 폴링으로 이미 표시된 메시지는 다시 표시되지 않음
