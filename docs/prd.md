# PRD — 오개념 교정 대화 시뮬레이터

## 1. 제품 개요
교사(사용자)가 오개념을 지닌 학생(챗봇)과 대화하고, 필요 시 튜터(챗봇)가 개입하여 대화를 고품질의 개념 중심 상호작용으로 이끌도록 설계된 웹 애플리케이션. 대화 로그와 교사의 질문을 분석하여 ‘교육 움직임’ 분류와 빈도 통계를 제공하고 CSV로 내보내기 지원.

본 서비스는 관리자 생성 “대화의 상황(시나리오)”을 기반으로 운영되며, 각 상황에는 학생의 오개념/특성, 문제 맥락, 활성화 여부가 포함된다. 분석 기준은 관리자 화면에서 사전 등록된 분석 프레임워크(예: 높은/낮은 레버리지 교육 움직임) 중 선택 적용한다.

## 2. 핵심 사용자 시나리오 (User Stories)
- **교사(사용자)**  
  1) 지정된 ‘대화의 상황’을 선택한다.  
  2) 학생 챗봇과 대화하며 질문을 던진다.  
  3) 대화 품질에 문제가 생기면 튜터 챗봇이 개입한다.  
  4) 세션 종료 시, 자신의 질문 유형(높은/낮은 레버리지)과 피드백 리포트를 확인하고 CSV로 다운로드한다.

- **관리자**  
  1) 여러 ‘대화의 상황’을 생성/수정/활성화한다.  
  2) 분석 기준(분석 프레임워크) 세트를 선택·적용한다.  
  3) 세션 로그 및 통계를 조회·다운로드한다.

- **학생 챗봇**  
  1) 시나리오에 정의된 오개념에 따라 반응한다.  
  2) 일정 기준(개념적 질문 누적 횟수) 충족 시 점진적 이해와 ‘아하 모먼트’를 드러낸다.

- **튜터 챗봇**  
  1) 대화 정체/탈선/낮은 레버리지 질문 감지 시 개입한다.  
  2) 더 높은 레버리지 질문을 제안하거나 긍정적 강화 피드백을 제공한다.

## 3. 핵심 기능 (Must-Haves)
- 3자 대화 인터페이스(학생·교사·튜터)와 역할 구분 라벨, 타임스탬프 처리
- 관리자용 시나리오 빌더(오개념·학생 특성·문제·활성화 토글)
- 분석 기준 선택(높은/낮은 레버리지 분류 체계) 및 자동 분류 리포트
- 세션 저장/조회/CSV 내보내기
- 최소 로그인(학생 고유번호 + 닉네임)

## 4. 성공 지표 (Success Metrics)
- 세션당 **높은 레버리지 질문 비율** 상승
- 세션 종료 비율/소요 시간
- ‘아하 모먼트’ 도달률
- 교사 재방문/다운로드 빈도

## 5. UX 요구사항
- 대화창에 역할별 말머리/아이콘과 타임스탬프 표시
- 튜터 개입은 **필요 시에만** 최소화
- 분석 리포트: 질문 목록의 유형 배지, 빈도 막대그래프, 요약 피드백
- 반응형 레이아웃, 다크 모드(선택)

## 6. 비기능 요구사항
- 서버: FastAPI (백엔드·템플릿 렌더), SQLite3
- 평균 지연 1초대(텍스트), 동시 접속 100명 목표
- 개인정보 최소화(로그인 2필드), CSV 익명 식별자
- 감사·추적: 관리자 변경 로그 기록

## 7. 제약 및 가정
- 분석 프레임워크는 관리자에 의해 사전 정의/선택
- 챗봇 및 분석 LLM은 분리된 모델(예: GPT-5 계열) 사용
- 파일/이미지 업로드 없이 텍스트 중심(초기 버전)

## 8. 릴리즈 범위
- v1: 시나리오 관리, 3자 대화, 분석/리포트, CSV
- v1.1: 다크 모드, 그래프 커스터마이즈
- v1.2: 시나리오 템플릿 갤러리, 태그 검색
